# Web — @sannai/web

## Stack

- **Framework**: TanStack Start (TanStack Router + Nitro SSR)
- **Build**: Vite 7, TypeScript 5
- **Styling**: Tailwind CSS v4 (via @tailwindcss/vite plugin, no postcss config)
- **State**: Zustand for client state, TanStack Router loaders for server data
- **Charts**: Recharts
- **Validation**: Zod

## Project Layout

```
src/
  routes/         # File-based routing — add file, route auto-registers
    __root.tsx    # Root layout (nav, head, shell)
    index.tsx     # /
    sessions.tsx  # /sessions
  components/     # Shared UI components
  utils/          # Helpers (seo.ts, etc.)
  styles/         # Tailwind CSS entry (app.css)
  router.tsx      # Router factory (getRouter)
  routeTree.gen.ts  # Auto-generated — DO NOT edit
```

## Commands

```bash
npm run dev        # Dev server on :3000
npm run build      # Production build (vite build + tsc)
npm run start      # Run production server (.output/server/index.mjs)
npm run typecheck  # Type check without emitting
npm run lint       # ESLint
```

## Conventions

- Path alias `~/` maps to `src/` (e.g. `import { seo } from '~/utils/seo'`)
- Routes are file-based: `src/routes/foo.tsx` creates `/foo`, `src/routes/foo.$id.tsx` creates `/foo/:id`
- `routeTree.gen.ts` is auto-generated by TanStack — gitignored, never edit manually
- API proxy: `/api/*` forwards to `http://localhost:8080` (Go backend) in dev
- Use `createFileRoute` for page routes, `createRootRoute` for the root layout

## Testing

Use https://github.com/vercel-labs/agent-browser for browser-based testing of the web frontend.

## Key APIs (from local agent on :9847)

These are available directly during development (until the Go cloud API is built):

- `GET /health` — status + version
- `GET /sessions` — list sessions (`?limit=20&offset=0`)
- `GET /sessions/:id` — session detail
- `GET /sessions/:id/events` — all events for a session

Start the agent with `make manual-test-live` or via:
```bash
SANNAI_CLAUDE_DIR=/path/to/test/data cargo run --manifest-path ../agent/Cargo.toml -- start
```
