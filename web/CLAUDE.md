# Web — @sannai/web

## Stack

- **Framework**: TanStack Start (TanStack Router + Nitro SSR)
- **Build**: Vite 7, TypeScript 5
- **Styling**: Tailwind CSS v4 (via @tailwindcss/vite plugin, no postcss config)
- **Data fetching**: TanStack Query (useQuery/useSuspenseQuery + route loaders via ensureQueryData)
- **Tables**: TanStack Table (headless, used for sessions list with sorting/filtering)
- **Forms**: TanStack Form + Zod adapter (available, not yet used in routes)
- **Client state**: TanStack Store (replaces zustand — see `utils/store.ts`)
- **Charts**: Recharts
- **Validation**: Zod (used for API response schemas in `utils/api.ts`)

## Commands

```bash
npm run dev        # Dev server on :3000
npm run build      # Production build (vite build + tsc)
npm run start      # Run production server (.output/server/index.mjs)
npm run typecheck  # Type check without emitting
npm run lint       # ESLint
```

## Data Flow Pattern

1. Route `loader` calls `context.queryClient.ensureQueryData(queryOptions)` — prefetches on navigation
2. Component uses `useSuspenseQuery(queryOptions)` — reads from cache, no loading spinners
3. Query option factories in `utils/queries.ts` define `queryKey` + `queryFn` pairs
4. API client in `utils/api.ts` validates responses with Zod schemas

## Conventions

- Path alias `~/` maps to `src/` (e.g. `import { seo } from '~/utils/seo'`)
- Routes are file-based: `src/routes/foo.tsx` creates `/foo`, `src/routes/foo.$id.tsx` creates `/foo/:id`
- `routeTree.gen.ts` is auto-generated by TanStack — gitignored, never edit manually
- Router context provides `queryClient` — access via `Route.useRouteContext()` or loader `context`
- API proxy: `/api/*` forwards to `http://localhost:8080` (Go backend) in dev
- Agent API fetched directly from `http://localhost:9847` (local agent)
- Use `createFileRoute` for page routes, `createRootRouteWithContext` for the root layout
- Prefer `useSuspenseQuery` over `useQuery` in route components (loaders guarantee data is cached)

## Key APIs (from local agent on :9847)

- `GET /health` — status + version
- `GET /sessions` — list sessions (`?limit=20&offset=0`)
- `GET /sessions/:id` — session detail
- `GET /sessions/:id/events` — all events for a session
